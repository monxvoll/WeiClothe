# User Story Backlog — Digital Closet

**Description:** This diagram contains the product backlog for the **Digital Closet** application. It outlines the core functionalities required for the initial development phases, including secure user authentication (IAM), smart image processing for automatic clothing identification, and fundamental inventory management features to help users digitize their wardrobes.

---

| Field | US-01 | US-02 | US-03 | US-04 | US-05 | US-06 | US-07 |
|---|---|---|---|---|---|---|---|
| **Epic / Module** | Access Management (IAM) | Access Management (IAM) | Access Management (IAM) | Smart Processing | Inventory Management | Inventory Management | Inventory Management |
| **User Story** | As a new user, I want to register by providing my personal data, so that I can have my own digital closet. | As a registered user, I want to log in with my email and password, so that I can manage my private inventory. | As an authenticated user, I want to log out securely, so that I can protect my information on shared devices. | As a user, I want to upload a photo of a garment, so that the system automatically identifies what type of clothing it is. | As an authenticated user, I want to save a garment in my closet with its attributes (color, type, size), so that I can digitize my wardrobe. | As a user, I want to see a dashboard with all my registered clothing, so that I can easily visualize my outfit options. | As a user, I want to be able to delete a garment from my closet, so that I can keep my digital inventory updated if I give away or throw out the physical clothes. |
| **Acceptance Criteria** | - Form requires: First Name, Last Name, Email, Date of Birth, Gender, and Password. <br> - The email must not previously exist in the system. <br> - The registration must communicate with the identity provider (Keycloak). <br> - Show a success message upon completion. | - Validate that the credentials are correct. <br> - The system must return a valid JWT token (400 if invalid). <br> - Block access if the data is incorrect. | - Invalidate the current JWT token (Endpoint `/api/wei/auth/logout`). <br> - Return code 204 No Content if successful. <br> - Redirect the user to the login screen. | - Allow file uploads in JPG or PNG format (suggested size limit: 5MB). <br> - Send the image to the Machine Learning service. <br> - Return the suggested label (e.g., "Shirt", "Pants") for the user to confirm. | - The endpoint must require a valid Bearer token (JWT) in the Headers. <br> - The garment must be strictly tied to the Learning ID (sub) of the user who uploads it. <br> - Save the photo path and metadata in the PostgreSQL database. | - The system must retrieve only the clothing that belongs to the authenticated user. <br> - Display the information in a grid with the main photo of each garment. <br> - Show a loading state if there are many garments. | - Request confirmation ("Are you sure you want to delete this?") before deleting. <br> - Validate that the user attempting to delete is the actual owner of the garment. <br> - Delete the record from the database and update the dashboard. |
